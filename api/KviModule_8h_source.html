163  // the user will be unable to unload the module unless he will specify
164  // the -f switch to the <module>.unload command. Without the -f switch

167  // The usage of this method is encouraged
168  // only when you have blocking dialogs inside the module code, like the
169  // QMessageBox or QFileDialog static methods.
170  // In this case you're entering a modal event loop that you can't control
171  // and if some script will attempt to forcibly unload the module
172  // it will surely lead to a crash when the static method returns (jumping into no mans land).
173  // <module>.unload -f is in fact undocumented so people will substantially
174  // not use it (unless they are developers and they are reading this comment).
175  //
